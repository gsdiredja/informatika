🛠️ Materi 3: Menemukan Eror (Debugging) & Mempercepat Program (Optimasi)

1. Apa itu Debugging dan Kenapa Penting?

Setelah berhasil membuat program di Materi 2, tugas seorang programmer belum selesai[cite: 2]. Terkadang, kode yang kita tulis tidak memunculkan pesan error berwarna merah, tetapi hasilnya aneh atau tidak masuk akal. Ini disebut Logical Error (Bug)[cite: 2].

Proses mencari dan memperbaiki bug ini dinamakan Debugging[cite: 2]. Mari kita bedah salah satu bug yang sering terjadi pada simulasi burung kita.

🪲 Studi Kasus Bug: Burung Menembus Tanah!
Jika kita membuat simulasi dengan perulangan waktu manual, komputer sering kali kebablasan menghitung. Hasil akhirnya, ketinggian burung (variabel y) bisa bernilai minus (misalnya -1.5 meter)[cite: 2]. Secara logika fisika ini salah, karena burung tidak bisa menembus bumi!

Cara Debugging-nya: Kita harus mengunci nilainya menggunakan logika if. Jika nilainya di bawah 0, paksa menjadi 0[cite: 2]. 
if y < 0:
    y = 0.0  # Burung berhenti di permukaan tanah![cite: 2]

2. Mengoptimasi Kode (Biar Program Nggak Lemot)

Bayangkan jika kita ingin membuat game Angry Birds sungguhan yang menghitung lintasan untuk 1.000 burung sekaligus. Jika kita menggunakan perulangan manual (seperti perintah while yang menghitung detiknya satu per satu), komputer akan bekerja sangat keras dan game bisa menjadi patah-patah (lag)[cite: 2].

Solusi optimasinya adalah menggunakan teknik Vectorization dengan bantuan pustaka numpy[cite: 2].

Cara Lama (Lambat) Cara Baru / Optimasi (Cepat)
Menggunakan perulangan while. Komputer harus menghitung detik 0.2, lalu simpan, lalu putar lagi ke detik 0.4, dan seterusnya[cite: 2]. Berulang-ulang. Menggunakan np.linspace()[cite: 2]. Komputer langsung membuat 100 titik waktu sekaligus dalam satu kali perintah, lalu rumusnya dihitung serentak oleh prosesor[cite: 2].
